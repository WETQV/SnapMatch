# SnapMatch: AI-бот для Telegram с MCP и режимом секретаря

SnapMatch стал публичным проектом. Это desktop-приложение для тех, кто хочет управлять Telegram-ботом с AI-моделями через понятную панель, без ручного редактирования конфигов под каждое изменение.

Через SnapMatch можно подключить своего Telegram-бота, добавить одну или несколько моделей, настроить поведение ответов, включить обработку голосовых сообщений, хранить историю и подключать внешние инструменты через MCP.

![Главные настройки](screenshots/main-settings.png)

Что умеет SnapMatch:

- управлять Telegram-ботом из desktop-панели;
- работать с OpenAI-compatible API, OpenRouter, Groq и локальными endpoints;
- переключать и балансировать несколько AI-моделей;
- подключать MCP-серверы и давать модели доступ к внешним инструментам;
- обрабатывать голосовые сообщения через Vosk, OpenAI Whisper или Groq;
- хранить историю сообщений, пользователей и событий;
- включать режим секретаря для черновиков, подтверждаемых ответов или автоответов;
- настраивать доступ, лимиты, системные промпты и поведение бота без ручной правки кода.

![MCP](screenshots/mcp-tab.png)

MCP здесь нужен для расширения возможностей бота. Это могут быть локальные скрипты, интеграции, файловые операции, свои сервисы или другие серверы, совместимые с Model Context Protocol. В интерфейсе можно добавить сервер, выбрать способ подключения, посмотреть доступные инструменты и ограничить доступ.

![Секретарь](screenshots/secretary-tab.png)

Режим секретаря помогает обрабатывать входящие сообщения аккуратнее: собрать несколько сообщений в один контекст, подготовить черновик, попросить подтверждение перед отправкой или ответить автоматически в выбранном сценарии. Это полезно для личного помощника, клиентских обращений, рабочих чатов и тестовых AI-сценариев вокруг Telegram.

Проект подойдёт тем, кто хочет собрать своего AI-помощника в Telegram, поэкспериментировать с MCP или держать управление ботом в отдельном desktop-приложении.
